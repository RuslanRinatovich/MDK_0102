## 🎯 Цель работы

Научиться использовать аннотации JUnit Jupiter (`@BeforeEach`, `@AfterEach`, `@BeforeAll`, `@AfterAll`) для создания изолированного тестового окружения и управления состоянием объектов перед выполнением тестов.

---

## 📚 Введение

В этой работе мы продолжим изучать тестирование. Вы уже знаете, как писать простые тесты с проверками (`Assertions`). Но часто бывает так, что перед каждым тестом нужно создавать новые объекты (как новых бойцов перед боем 🥊), а после теста — например, закрывать файлы или очищать данные. 🧹

Для этого в JUnit есть специальные методы, которые помечаются аннотациями:

| Аннотация | Описание | Когда использовать |
|:---------:|----------|-------------------|
| `@BeforeAll` 🚀 | Выполняется **один раз** перед **всеми** тестами. Метод должен быть `static`. | Тяжелая инициализация (подключение к БД, загрузка настроек) |
| `@BeforeEach` 🔄 | Выполняется **перед каждым** тестом. | Создание "свежих" объектов для изоляции тестов |
| `@AfterEach` 🧹 | Выполняется **после каждого** теста. | "Уборка" после каждого теста (очистка файлов, сброс счетчиков) |
| `@AfterAll` 🏁 | Выполняется **один раз** после **всех** тестов. Метод должен быть `static`. | Закрытие ресурсов (соединений с БД, файлов) |

## 💡 Пример выполнения (Образец)

Рассмотрим класс `BankAccount` (Банковский счет). 🏦

```java
// Класс BankAccount (предоставляется преподавателем)
public class BankAccount {
    private String owner;
    private double balance;

    public BankAccount(String owner, double initialBalance) {
        this.owner = owner;
        this.balance = initialBalance;
    }

    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
        }
    }

    public boolean withdraw(double amount) {
        if (amount > 0 && balance >= amount) {
            balance -= amount;
            return true;
        }
        return false;
    }

    public double getBalance() {
        return balance;
    }
    
    public void resetBalance() {
        this.balance = 0.0;
    }
}
```

**Задача:** Написать тесты для класса `BankAccount`, используя аннотации жизненного цикла, чтобы убедиться, что тесты не влияют друг на друга. ✍️

**Решение:**

```java
import org.junit.jupiter.api.*;
import static org.junit.jupiter.api.Assertions.*;

public class BankAccountTest {

    private BankAccount account;

    @BeforeAll
    static void initAll() {
        System.out.println("🔧 Запуск всех тестов для BankAccount. Инициализация общих ресурсов...");
    }

    @BeforeEach
    void setUp() {
        // Создаем НОВЫЙ счет перед КАЖДЫМ тестом
        account = new BankAccount("Иван Петров", 1000.0);
        System.out.println("  ✅ Создан новый счет для теста. Баланс: 1000.0");
    }

    @AfterEach
    void tearDown() {
        // "Обнуляем" счет или просто логируем завершение теста
        System.out.println("  🧹 Тест завершен. Очистка не требуется, объект будет удален сборщиком мусора.");
    }

    @AfterAll
    static void tearDownAll() {
        System.out.println("🏁 Все тесты BankAccount завершены. Освобождение общих ресурсов.");
    }

    @Test
    @DisplayName("Тест: Пополнение счета увеличивает баланс 📈")
    void testDeposit() {
        double balanceBefore = account.getBalance();
        account.deposit(500.0);
        assertEquals(balanceBefore + 500.0, account.getBalance(), 0.001);
        System.out.println("    ⚔️ Выполняется testDeposit");
    }

    @Test
    @DisplayName("Тест: Снятие средств уменьшает баланс 📉")
    void testWithdrawSuccess() {
        account.withdraw(300.0);
        assertEquals(700.0, account.getBalance(), 0.001);
        System.out.println("    ⚔️ Выполняется testWithdrawSuccess");
    }

    @Test
    @DisplayName("Тест: Снятие средств больше баланса невозможно 🚫")
    void testWithdrawFail() {
        boolean result = account.withdraw(1500.0);
        assertFalse(result);
        assertEquals(1000.0, account.getBalance(), "Баланс не должен измениться");
        System.out.println("    ⚔️ Выполняется testWithdrawFail");
    }
}
```

**Что здесь происходит?** 🤔

1.  `@BeforeAll` запускается **1 раз** в самом начале. 🚀
2.  Перед **первым** тестом (`testDeposit`) срабатывает `@BeforeEach` и создает счет с 1000 рублей. ✅
3.  Выполняется `testDeposit`. ⚔️
4.  После первого теста срабатывает `@AfterEach`. 🧹
5.  Перед **вторым** тестом (`testWithdrawSuccess`) снова срабатывает `@BeforeEach` и создает **новый** счет с 1000 рублей. 🔄
6.  Выполняется `testWithdrawSuccess`. ⚔️
7.  И так далее для каждого теста.
8.  В самом конце, после всех тестов, срабатывает `@AfterAll`. 🏁

✅ Это гарантирует, что каждый тест работает со своим, **"свежим"** объектом, и результат одного теста **не влияет** на другой.
